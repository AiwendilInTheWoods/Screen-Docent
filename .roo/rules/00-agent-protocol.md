# Project Context Protocol
Before taking ANY action or writing ANY code, you MUST execute the following protocol:
1. Read `.ai/active_context.md` to understand the current task and state.
2. Read `.ai/system_architecture.md` to ensure your proposed solution aligns with the tech stack.

# Execution Protocol
1. Write the code or execute the terminal commands.
2. Run standard tests or syntax checks.

# Wrap-Up Protocol
When the task is complete and I have approved the changes, you MUST:
1. Update `.ai/active_context.md` to reflect the new state of the project.
2. If a major architectural decision was made or a library was chosen, append it to `.ai/decision_log.md`.
